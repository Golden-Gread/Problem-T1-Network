# 服务器连接

#### 时间限制：1000ms      内存限制：512MB

## 题目描述

在我们日常生活中，我们经常需要进行服务器连接，比如银行卡绑定、支付宝绑定等。

假设我们有若干台服务器，每台服务器都有自己的IP地址S与后缀表达式E，我们希望能够通过IP地址来连接到对应的服务器。

IP地址的格式为“%d.%d.%d.%d”，且%d的取值范围为0-255。


在连接服务器成功后，我们会获得来自服务器的后缀表达式E，我们需要计算E的和，并将结果返回给用户。

例如，假设有两台服务器，分别有IP地址192.168.1.1和192.168.1.2，后缀表达式分别为“512+4*+3-”和“931-3*+1+”。
      我们通过IP地址192.168.1.1来连接到第一台服务器，并计算后缀表达式“512+4*+3-”，得到结果为14。

现在，我们希望设计一个程序，能够根据IP地址快速找到对应的服务器，并能够快速连接到该服务器并获取信息。


### 输入格式

输入的第一行包含一个整数n，表示操作个数
接下来n行，依次描述每个操作，每行第一个字符串 op 表示操作类型：
```
    若 op = "create", 表示创建一个服务器，后面跟着IP地址S和后缀表达式E，用空格隔开。若该IP地址已存在或IP地址格式错误则返回failed，否则创建该服务器并返回success.

    若 op = "connect", 表示连接到一个服务器，后面跟着IP地址S，若该IP地址不存在或IP地址格式错误则返回failed，否则连接到该服务器并返回该服务器后缀表达式E的结果。

    若 op = "delete", 表示删除一个服务器，后面跟着IP地址S，若该IP地址不存在或IP地址格式错误则返回failed，否则删除该服务器并返回success。

    若 op = "modify", 表示修改一个服务器的后缀表达式，后面跟着IP地址S和新的后缀表达式F，用空格隔开，若该IP地址不存在或IP地址格式错误则返回failed，否则修改该服务器的后缀表达式并返回success。

```

### 输出格式

每行输出一个字符串，表示对应的操作的结果。

### 样例输入
```
6
create 192.168.1.1 512+4*+3-
create 192.168.1.2 931-3*+12/+
create 192.168.1.257 123+456-789*
modify 192.168.1.2 123+456-789*
connect 192.168.1.1
delete 192.168.1.3
```

### 样例输出
```
success
success
failed
success
14
failed
```

### 说明/提示

保证本题目中，IP地址的格式为“%d.%d.%d.%d”，后缀表达式的格式为“+*-”的组合且数字只为0-9。

n<=10^6，操作类型为op=create/connect/delete，IP地址长度不超过30，后缀表达式长度不超50。
